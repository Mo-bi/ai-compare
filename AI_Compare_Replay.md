# AI Compare 开发复盘记录 (2026-03-13)

这份文档总结了本次协作中关于 **AI Compare** 综述模块重构、持久化存储及流式 API 实现的全过程，旨在为后续维护和功能扩展提供参考。

---

## 1. 初始目标与环境搭建
- **扩展安装**：安装了 `conductor`（工作流管理）和 `superpowers`（工程技能包）。
- **任务定义**：彻底重构已失效的综述模块，实现多窗口历史读取、筛选、API 综述及一键复制功能。

## 2. 核心功能演进

### A. 内容提取算法 (Scraping Strategy)
这是本次开发中最具挑战性的部分，经历了四个阶段的迭代：
1. **语义选择器阶段**：通过特定的 CSS 类名（如 `.message`）抓取，但因 AI 官网类名混淆且频繁变动而失效。
2. **坐标位置判定阶段**：通过元素在屏幕上的左右位置判定角色（右侧为用户，左侧为 AI）。解决了大部分国产模型的适配问题。
3. **针对性适配阶段**：针对 **智谱清言 (ChatGLM)** 这种不分左右、靠名字标识（如 "neo"）的特殊布局进行了专项优化。
4. **末端节点去重法 (Leaf-Node Only)**：为了解决 HTML 嵌套导致的“同一段话重复 6 次/4 次”的问题，引入了物理包含判断逻辑——如果元素 A 包含元素 B，则彻底丢弃父级 A，仅保留最深层的叶子节点 B。

### B. 提问追踪架构优化 (Architecture)
- **从“抓取”转向“记录”**：放弃从网页 DOM 中“盲猜”用户的提问（容易被名字、侧边栏干扰）。
- **系统级记录**：在全局输入框发送 (handleSend) 阶段，由系统 100% 准确记录提问内容并存入 `userQueries`。
- **纯净拼接**：综述时，使用“系统记录的准确提问”+“网页抓取的 AI 回答”，确保了上下文的绝对纯净。

### C. 数据持久化 (Persistence)
- **存储迁移**：将 Prompt 模板从不稳定的浏览器 `localStorage` 迁移到 Electron 的系统级数据目录 (`userData`)。
- **重装不丢数据**：模板现在保存在 `~/Library/Application Support/ai-compare/summary_prompts.json`，确保软件更新或卸载重装后，用户自定义模板依然存在。

### D. 流式 API 模式 (Streaming API)
- **主进程代理**：通过 Electron 主进程发起 API 请求，彻底绕过了渲染进程的跨域 (CORS) 限制。
- **IPC 流式传输**：实现了 `api:proxy-stream` 通道，支持 Chunk 级的实时数据推送。
- **数据缓冲区**：在前端引入 Buffer 机制处理 SSE 分包，实现了平滑的打字机输出效果。

## 3. 关键问题修复 (Bug Fixes)
- **黑屏问题**：修复了由于 `electron.d.ts` 缺失接口定义导致的 TypeScript 运行时崩溃。
- **Kimi 发送失败**：针对 Kimi 的 `contenteditable` 输入框，改用 `execCommand('insertText')` 模拟真实用户输入，解决了 Kimi 无法识别文本并激活发送按钮的问题。
- **智谱角色反转**：通过限制名字识别范围（仅限顶部 100 字符），防止 AI 在正文中提到用户名字时导致的身份误判。

## 4. 最终产物
- **打包版本**：成功构建了 macOS (Intel & Apple Silicon) 双架构 `.dmg` 安装包。
- **专家 Prompt**：预置了“高级 AI 智库综述专家”模板，具备极强的抗噪和深度分析能力。

## 5. 后续维护建议
- **网页改版应对**：若某模型抓不到内容，优先检查 `WebviewPanel.tsx` 中的 `getHistoryScript` 过滤死区（left 15%）是否需要调整。
- **API 扩展**：若需增加新模型 API，只需在 `App.tsx` 的 `endpoints` 对象中增加对应的 URL 和 Model 映射。
